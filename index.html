§1 - HTML

  • HTML - відповідає за наповнення веб-сторінки контентом, структуру і зміст.
  • CSS - використовується для оформлення та позиціонування, зовнішній вигляд.
  • JavaScript - дозволяє додати функціонал, реакцію на дії користувача.

Специфікація HTML Living Standard - головний документ, який описує стандарти, можливості і майбутній розвиток мови HTML.

Довідники:

  • Довідник HTML-тегів Для того щоб дізнатися доступні атрибути тегу та їх призначення

  • Довідник CSS-властивостей
  • Керівництво для написання коду в процесі вивчення технологій.
  1. <p> - універсальний контейнер для групування дрібних фразових елементів, відокремлення їх один від одного, і подальшої стилізації.
  2. <h1> - <h6> - визначає текстові заголовки смислових розділів різного рівня, які вказують на важливість секції контенту, розташованого після них.
  3. <li> - елементи списку, діти списку
  4. <ol> - створює нумерований (упорядкований) список, тобто кожен елемент списку пронумерований.
  5. <ul> - створює маркований (невпорядкований) список, кожен елемент якого починається з невеликого символу (маркера).
  6. <a> - призначений для створення посилань, тексту, клікаючи на який, переходимо на іншу сторінку, завантажуємо файл тощо.
    • href="адреса" - aдреса посилання задається в обов'язковому атрибуті
    • target="_blank" - визначає те, в який вкладці повинен відкриватися документ, на який веде посилання.
    • rel="noreferrer nofollow noopener" - відношення між поточним і пов'язаним документом
    • download="ім'я файлу" - повідомляє браузеру, що зазначений ресурс не повинен відкриватися, а завантажуватися в момент, коли користувач клікне на посилання.
    • href="tel:+14251234563" - здійснювати дзвінки на телефони
    • href="mailto:example@mail.ua - відправляти повідомлення
  7. <button> - Інтерактивний елемент, який оживляється за допомогою JavaScript.
    • type="button"
    • type="submit" - за замовчуванням
  8. <img> - призначений для розмітки зображень у різних графічних форматах.
    • src="шлях" - обов'язковий атрибут, вказує адресу зображення. Шлях до зображення може бути абсолютним, або відносним.
    • alt="опис" - обов'язковий атрибут, альтернативний опис.
    • width="значення" і height="значення" - задають розміри зображення в пікселях.
  9. <header> - Шапка (хедер) виділяє вступну частину всієї сторінки, складної секції або статті.
  10. <footer> - Підвал (футер) - це фінальна частина всієї сторінки, її розділу або статті.
  11. <main> - Основний (унікальний) вміст, що не повторюється на інших сторінках сайту.
  12. <nav> - Визначає розділ основної навігації з посиланнями на секції поточної або інших сторінок.
  13. <article> - Незалежна, відокремлювана, і така, що має самостійне значення, частина документа.
  14. <section> - Великий розділ, що об'єднує вміст за смисловим значенням.
  15. <div> - Універсальний контейнер без семантичного значення.
  16. <table> - визначає всю таблицю, контейнер для контенту.
  17. <tr> - рядок таблиці (table row). Не може бути порожнім, повинен містити хоча б одну клітинку.
  18. <td> - клітинка таблиці (table data). Можна використовувати тільки всередині рядка.
  19. <th> - (table header) - семантичний тег клітинки-заголовка.
  20. <thead> - шапка, обгортає групу рядків таблиці в області заголовка.
  21. <tbody> - тіло, обгортає основну групу рядків таблиці.
  22. <tfoot> - підвал, означає групу рядків таблиці в області нижнього колонтитула.
  23. <caption> - заголовок таблиці
    Example
    Cloud service Statistics
    Users Visits Volume
    Google 12000 307000 1000 Tb
    Amazon 9000 200500 800 Tb
  24. <dl> - це контейнер для списку пар термінів та їх описів, тобто словник.
  25. <dt> - для розмітки термінів
  26. <dd> - для описів
  27. <span> - Універсальний контейнер для дрібного текстового контенту
  28. <pre> - дозволяє 
                додати переформатований текст 
                з примусовими перенесеннями рядків.
  29. <sup> і <sub>- для відображення символів у верхньому (sup, скорочено від superscript) і нижньому (sub, скорочено від subscript) індексі.
  30. <time> - для часу datetime="pppp-мм-дд" - атрибут - 14 грудня 2019 року.
  31. <address> - для зберігання контактної інформації про автора
  32. <strong> і <b> - роблять текст жирним <strong> - семантичний
  33. <em> і <i> - роблять текст курсивним <em> - семантичний
  34. <video> - Для розмітки відеоконтенту
    • src - вказує адресу відеофайлу
    • poster - адреса зображення, яке використовується як постер, якщо відео завантажене або ще не відтворюється
    • width і height - задають висоту і ширину як у зображення
    • controls - атрибут-прапорець, якщо присутній, додаються елементи управління відео
    • autoplay - атрибут-прапорець, якщо присутній, відтворення відео починається як тільки воно доступне
    • loop - атрибут-прапорець, якщо присутній, відео буде повторюватися в циклі
    • preload - вказує режим попереднього завантаження відео
    • none - не завантажувати відео заздалегідь
    • metadata - завантажити службову інформацію (тривалість тощо)
    • auto - заздалегідь завантажувати всі відео
  35. <audio> - Для розмітки аудіоконтенту
    • src - вказує адресу відеофайлу
    • controls - атрибут-прапорець, якщо присутній, додаються елементи управління аудіо
    • autoplay - атрибут-прапорець, якщо присутній, відтворення аудіо починається як тільки воно доступне
    • loop - атрибут-прапорець, якщо присутній, аудіо буде повторюватися в циклі
    • preload вказує режим попереднього завантаження аудіофайлу
    • none - не завантажувати аудіо заздалегідь
    • metadata - завантажити службову інформацію (тривалість тощо)
    • auto - заздалегідь завантажувати увесь аудіофайл
